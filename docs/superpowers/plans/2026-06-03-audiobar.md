# AudioBar Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a small macOS menu-bar app that lists active CoreAudio output processes and exposes honest per-app volume sliders for known scriptable media apps.

**Architecture:** `AudioBarCore` owns process models, CoreAudio enumeration, AppleScript volume support, and formatting. The executable target owns the SwiftUI `MenuBarExtra`, refresh store, and app lifecycle. A project-local script builds a SwiftPM app bundle and launches it.

**Tech Stack:** SwiftPM, Swift 6, SwiftUI `MenuBarExtra`, AppKit, CoreAudio, XCTest.
