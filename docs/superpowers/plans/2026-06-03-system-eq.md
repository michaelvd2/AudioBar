# System EQ Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a compact 10-band EQ tuner to AudioBar and wire it to an explicit system-audio engine boundary.

**Architecture:** `AudioBarCore` owns testable EQ settings, presets, and engine status. The menu-bar app owns SwiftUI controls and persistence through `AudioProcessStore`. The first runtime engine slice exposes CoreAudio tap readiness/status truthfully so the UI never claims active EQ without an active system route.

**Tech Stack:** SwiftPM, Swift 6, SwiftUI, CoreAudio, XCTest.
